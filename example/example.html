Is EtherCSS for you?

You should use EtherCSS if you're embarking on a smaller project or just don't feel like you need all the utility of larger frameworks. EtherCSS only styles a handful of standard HTML elements and includes a grid, but that's often more than enough to get started. In fact, this site is built on EtherCSS and has ~200 lines of custom CSS (half of which is the docking navigation).

The grid

The grid is a 12-column fluid grid with a max width of 960px, that shrinks with the browser/device at smaller sizes. The max width can be changed with one line of CSS and all columns will resize accordingly. The syntax is simple and it makes coding responsive much easier. Go ahead, resize the browser.

Typography

Type is all set with the rems, so font-sizes and spacial relationships can be responsively sized based on a single <html> font-size property. Out of the box, EtherCSS never changes the <html> font-size, but it's there in case you need it for your project. All measurements are still base 10 though so, an <h1> with 5.0remfont-size just means 50px.

The typography base is cross-browser compatible sans-serif, set at 1rem (16px) over a 1.6 line height (25.6px). Other type basics like anchors, strong, emphasis, and underline are all also included.

Headings create a family of distinct sizes each with specific letter-spacing, line-height, and margins.

Buttons

Buttons come in two basic flavors in EtherCSS. The standard <button> element is plain, whereas the .button-primary button is vibrant and prominent. Button styles are applied to a number of appropriate form elements, but can also be arbitrarily attached to anchors with a .button class.

Code

Code styling is kept basic – just wrap anything in a <code> and it will appear like this. For blocks of code, wrap a <code> with a <pre>.


.some-class {
  background-color: red;
}



<pre><code>.some-class {
  background-color: red;
}</code></pre>

<!-- Remember every whitespace and break will be preserved in a <pre>,
including indentation in your code -->

Media queries

EtherCSS uses media queries to serve its scalable grid, but also has a list of queries for convenience of styling your site across devices. The queries are mobile-first, meaning they target min-width. Mobile-first queries are how EtherCSS's grid is built and is the preferrable method of organizing CSS. It means all styles outside of a query apply to all devices, then larger devices are targeted for enhancement. This prevents small devices from having to parse tons of unused CSS. The sizes for the queries are:

  • Desktop Ultra HD (TV): 1440px
  • Desktop HD: 1200px
  • Desktop: 992px
  • Tablet: 768px
  • Phablet: 544px
  • Mobile: 400px

/* Mobile first queries variables */

/* Larger than mobile */

$screen--xm: 400px;

/* Larger than phablet */

$screen--sm: 544px;

/* Larger than tablet */

$screen--md: 768px;

/* Larger than desktop */

$screen--lg: 992px;

/* Larger than Desktop HD */

$screen--xl: 1200px;

/* Larger than Desktop Ultra HD (TV) */

$screen--xxl: 1440px;

Utilities

EtherCSS has a number of small utility classes that act as easy-to-use helpers. Sometimes it's better to use a utility class than create a whole new class just to float an element.


/* Utility Classes */

/* Float either direction */

.ether-pull-right {
  float: right;
}

.ether-pull-left {
  float: left;
}

/* Clear a float */

.ether-clearfix {
  content: "";
  display: table;
  clear: both;
}
